# Bayti Recommendation Engine

This module contains the offline content-based recommender for the Bayti real estate project.

The recommender compares properties using cleaned structured fields, engineered real-estate features, encoded categorical values, and cosine similarity. It is designed to generate precomputed recommendations that the backend can store and return directly.

## Current Workflow

For the current scraped dataset:

```text
data/apartments.csv
-> legacy data cleaning
-> feature engineering
-> preprocessing
-> cosine similarity
-> data/precomputed_recommendations.csv
```

For future database/user-added data:

```text
database export
-> strict schema validation
-> feature engineering
-> preprocessing
-> cosine similarity
-> refreshed recommendation output
```

## Important Modes

The pipeline supports two modes:

| Mode | Purpose |
| --- | --- |
| legacy | Used for the current scraped CSV, because it contains missing values that the cleaning pipeline recovers. |
| strict | Used for future database exports and user-added data. The data must follow the backend schema before the recommender runs. |

## Main Files

| File | Purpose |
| --- | --- |
| `ML/data_loader.py` | Loads property data from CSV for now. Later this can point to a database export. |
| `ML/data_validation.py` | Validates strict incoming database/user data. |
| `ML/data_cleaning.py` | Cleans the legacy scraped dataset and creates `Final_price`. |
| `ML/feature_engineering.py` | Adds recommender features such as `Price_per_sqm`, location parts, amenity flags, salons, and master bedrooms. |
| `ML/data_preprocessing.py` | Scales numeric features, one-hot encodes categorical features, and builds the model-ready matrix. |
| `ML/recommendation_engine.py` | Computes similarity and retrieves similar apartments. |
| `ML/pipeline.py` | Orchestrates the full pipeline in `legacy` or `strict` mode. |
| `ML/recommendation_export.py` | Builds and saves backend-ready recommendation rows. |
| `ML/run_recommendation_batch.py` | Command-line runner for regenerating recommendations. |

## Generate Recommendations

Run this command from the project root:

```powershell
python -m recommender_system.ML.run_recommendation_batch
```

Default behavior:

```text
input:  recommender_system/data/apartments.csv
output: recommender_system/data/precomputed_recommendations.csv
mode:   legacy
top_n:  5
```

For a future strict database export:

```powershell
python -m recommender_system.ML.run_recommendation_batch --input path/to/database_export.csv --mode strict --require-identifier
```

## Recommendation Output

The generated file is:

```text
recommender_system/data/precomputed_recommendations.csv
```

Output columns:

```text
source_apartment_id
recommended_apartment_id
rank
similarity_score
```

The backend should import this output into a recommendations table and query it by `source_apartment_id`.

## Backend Contracts

The backend team should use these docs:

| Document | Purpose |
| --- | --- |
| `docs/property_schema.md` | Defines the property fields required by the recommender. |
| `docs/recommendation_contract.md` | Defines the recommendation table, query pattern, and refresh flow. |

Important backend rules:

- `apartment_id` is generated by the database, not entered by users.
- `URL` is optional metadata and can be null.
- `Sale_price` is required for sale listings.
- `Price_annualy` is required for rent listings.
- `Price_monthly` is optional and is not required by the current recommender.
- New user-added properties must follow the strict schema before recommendation refresh.

## Refresh Strategy

The recommender is offline. Recommendations are not recomputed on every website request.

Recommended refresh flow:

```text
latest full property inventory from database
-> run batch command in strict mode
-> generate refreshed recommendations
-> backend replaces/imports recommendation table
```

Use the latest full property inventory during refresh, not only newly added properties. New properties can become recommendations for existing properties too.

Recommended schedule:

```text
Run weekly, or whenever a large batch of properties is added or changed.
```

## Integration Checklist

- Store scraped apartment records in the website database.
- Preserve stable `apartment_id` values.
- Import `data/precomputed_recommendations.csv` for the initial scraped dataset.
- Create a backend table with `source_apartment_id`, `recommended_apartment_id`, `rank`, and `similarity_score`.
- Return recommendations by querying `source_apartment_id` and ordering by `rank`.
- Join `recommended_apartment_id` with the properties table to return display fields to the frontend.
- For future updates, export the latest full database inventory and rerun the batch command in `strict` mode.
